# Real Data Checklist

Use this checklist when replacing placeholder data with safe real information. Do not enter sensitive private data until the site has an agreed protection layer.

## Before Large Real Data Entry

- [ ] Backup/export local MySQL before large real data entry.
- [ ] Confirm no screenshots, PDFs, receipts, DB exports, or private files are staged.
- [ ] Use page UI for shared CRUD data instead of editing seed data after DB has been seeded.
- [ ] Confirm traveler display names are safe to show.
- [ ] Recheck public emergency / embassy / map links before the trip.

## Dashboard

- [ ] Replace placeholder trip deadline with the next real planning deadline.
- [ ] Replace placeholder city notes with the confirmed city sequence.
- [ ] Confirm the displayed traveler count is still correct.
- [ ] Update high-level booking statuses after each major booking decision.
- [ ] Keep emergency quick access limited to public numbers and safe summaries.

## Itinerary

- [ ] Replace placeholder daily titles with confirmed daily plans.
- [ ] Add real travel dates and city names.
- [ ] Add public attraction, station, airport, and restaurant names.
- [ ] Add safe transport notes without private ticket numbers.
- [ ] Add rough cost estimates as reference only.
- [ ] Add Google Maps search queries for public locations.
- [ ] Avoid hotel room details, confirmation numbers, and identity document reminders.

## Bookings

- [ ] Replace placeholder booking titles with real booking summaries.
- [ ] Add booking category, date, location summary, owner, status, and reference amount.
- [ ] Add real hotel name only if the group accepts the privacy risk; avoid confirmation number.
- [ ] Add booking amount as reference only.
- [ ] Create linked expense only after confirming payment.
- [ ] Do not paste full private booking URLs unless access control is verified.
- [ ] Do not add ticket numbers, QR codes, or complete confirmation numbers.

## Budget / Expenses

- [ ] Add real expense only after payment is confirmed.
- [ ] Select payer using the existing traveler IDs/display names.
- [ ] Select all split travelers.
- [ ] Confirm currency is EUR, SGD, or MYR.
- [ ] Mark settled only after repayment is complete.
- [ ] Link expense to itinerary or booking when relevant.
- [ ] Keep receipts, card details, and transfer references outside the repo.

## Packing

- [ ] Replace generic packing items with the group's real checklist.
- [ ] Assign shared items to the responsible traveler.
- [ ] Set per-traveler status to required, packed, or not needed.
- [ ] Add quantities for shared items where useful.
- [ ] Keep passport as an item only; do not enter passport numbers.
- [ ] Avoid detailed private medical information.

## Documents

- [ ] Keep placeholder document links until private storage permissions are verified.
- [ ] Confirm Google Drive, iCloud, OneDrive, or other folder permissions are limited to the intended travelers.
- [ ] Use a Documents access code only for protected folder links that need an extra app-level gate.
- [ ] Do not write the access code in notes, labels, screenshots, commits, or chat logs.
- [ ] Add safe labels for flight, hotel, train, attraction, insurance, and receipt folders.
- [ ] Do not enter passport number, card number, full policy number, or complete confirmation number.
- [ ] Do not upload or commit PDFs, screenshots, QR codes, receipts, passport scans, insurance certificates, or booking files.
- [ ] Do not enter full confirmation numbers, passport numbers, card numbers, or bank details into the app.
- [ ] Confirm the Documents warning is still visible on desktop and phone.

## Food

- [ ] Replace restaurant placeholders with real shortlist names.
- [ ] Add city, cuisine, priority, price range, and reservation status.
- [ ] Add public Google Maps search or place links.
- [ ] Avoid private reservation IDs or personal phone numbers.
- [ ] Test the Maps links on a phone.

## Attractions

- [ ] Replace attraction placeholders with confirmed shortlist items.
- [ ] Mark ticket requirement and booking status.
- [ ] Add estimated duration and official website when useful.
- [ ] Add public Google Maps search or place links.
- [ ] Avoid ticket barcodes, QR codes, and full booking references.

## Emergency Quick Access

- [ ] Keep Italy public emergency numbers visible.
- [ ] Add only public embassy or consular assistance contacts.
- [ ] Add generic nearby medical/pharmacy search links.
- [ ] Do not enter traveler personal phone numbers yet.
- [ ] Do not enter bank card numbers, insurance policy numbers, or hotel room details.
